# Yoyo KOC Exchange — 产品需求文档 (PRD)

**版本**: v1.0
**日期**: 2026-07-17
**状态**: 已上线迭代中
**产品定位**: 轻量级创作者（KOC）激励管理工具，替代谷歌表单+Excel人工结算流程

---

## 一、产品背景

### 1.1 痛点
- 创作者作品收集依赖谷歌表单 → 人工复制到 Excel → 手动计算互动数/积分
- 40+ 作者、600+ 链接/月，人工审核计算消耗大量时间
- 奖励兑换（周边/钻石/谷歌卡）依赖手工登记，容易出错
- 创作者触达依赖 Discord 私信，单人操作存在封号风险

### 1.2 目标
- 替代人工算分流程，实现"提交 → 算分 → 兑换"自动化
- 降低运营人力成本，提升创作者体验

---

## 二、系统架构

### 2.1 技术栈
- **前端**: 纯 HTML/CSS/JS (SPA，单文件部署)
- **后端**: Supabase (PostgreSQL + REST API)
- **部署**: GitHub Pages (自动部署)
- **机器人**: Discord Mochi Bot (私信广播)
- **算分**: OpenAI Codex Agent (浏览器扫描链接获取互动数)

### 2.2 两个端

```
┌─────────────────────────┐    ┌──────────────────────────────┐
│   创作者端 (Creator)      │    │   管理端 (Admin)              │
│   index.html             │    │   admin.html                 │
├─────────────────────────┤    ├──────────────────────────────┤
│  · 注册/登录              │    │  · KOC信息库 (台账+等级)      │
│  · 提交作品链接 (2次机会)  │    │  · 作品审核                   │
│  · 查看提交记录+进度提示   │    │  · 算分导入+预览+确认应用      │
│  · 兑换奖励 (钻石/卡/周边) │    │  · 积分计算 (月度)            │
│  · 查看等级徽章            │    │  · 周边发货管理               │
│  · FAQ                   │    │  · 端内奖励管理               │
│                         │    │  · 谷歌卡管理                 │
│                         │    │  · 规则配置 (可编辑阈值)        │
│                         │    │  · Mochi广播 (4阶段)           │
└─────────────────────────┘    └──────────────────────────────┘
```

## 三、用户角色

### 3.1 创作者 (Creator)
| 角色 | 说明 |
|------|------|
| 新创作者 | 首次进入需填写完整注册信息 (discord/account ID/game UID/姓名/服务器/地址/电话) |
| 老创作者 | 已有记录者，用 Discord + UID 直接登录 |
| 等级体系 | 认证创作官 (默认) → 金级创作官 → 铂金创作官 |

**新人月逻辑**: 入群的下个月为首个新人活动月，当月满5分额外+2分

### 3.2 管理员 (Admin)
- 单个管理员，密码保护 (`yoyo2026`)
- 全功能操作: KOC管理 / 作品审核 / 算分导入 / 奖励管理 / 广播 / 规则配置

## 六、积分规则

### 6.1 算分标准 (管理端可配置阈值)

| 平台 | 1分 | 2分 |
|------|-----|-----|
| TikTok | 200 views | 400 views |
| YouTube | 200 views + 20s 观看 | 400 views |
| Pinterest/FB/IG/X/Reddit | 40 engagements | 80 engagements |

### 6.2 积分上限
- **月上限**: 20 分
- **超过20未到40**: 按20算 (超出不计)
- **达到40分以上**: 按40全额结算

### 6.3 新人任务 (New Creator Bonus)
- 入群的下个月为新人月 (如7月16日入群 → 新人月为8月)
- 新人月内投稿满 **5 积分** → 当月结算时额外 +2 积分
- 老创作者标记 (old_creator) 跳过此任务
- 在 notes 中以 `newbie_month:YYYY-MM` 和 `newbie_bonus:YYYY-MM` 标记

### 6.4 连续创作奖励 (Consistent Creation - CC)
- 月投 **15+ 条** (任意平台)
- 奖励: **Premium Monthly Card x1 + Shelly Coins x500 + Gold x1000**
- 自动检测，在确认应用积分时自动创建端内奖励订单
- 在 notes 中以 `CC:YYYY-MM` 标记

### 6.5 等级体系

| 等级 | 晋升条件 | 徽章样式 |
|------|---------|---------|
| ✅ 认证创作官 | 默认等级 | 绿色渐变 |
| 🟡 金级创作官 | 连续2个月达到5分 | 金色渐变 |
| 💎 铂金创作官 | 连续3个月达到5分 | 紫色渐变 |

## 七、数据库模型

### kocs — 创作者信息表
```
uid            text PK       # YOYO-001 格式
discord_name   text          # Discord昵称
name           text          # 真实姓名
account_id     text          # 6位数字ID
region         text          # 地区
server         text          # 正式服 / 灯塔服
address        text          # 完整收货地址
tier           text          # certified / gold / platinum
status         text          # active / inactive
notes          text          # 标签: CC:YYYY-MM, old_creator, newbie_month:YYYY-MM, newbie_bonus:YYYY-MM
created_at     timestamptz   # 创建时间
```

### submissions — 作品提交表
```
id                     int8 PK       # 自增
uid                    text FK       # 关联 kocs
discord_name           text          # 冗余
server                 text          # 正式服/灯塔服
links_engagement       text          # 链接 (换行符分隔)
status                 text          # pending / scored
points_earned          int4          # 算得的积分
score_details          jsonb         # 详细算分数据
total_engagement_count int4          # 总互动数
total_view_count       int4          # 总浏览数
scored_at              timestamptz   # 评分时间
admin_notes            text          # 备注
created_at             timestamptz   # 提交时间
```

### point_logs — 积分流水表
```
id             int8 PK       # 自增
uid            text FK       # 关联 kocs
change         int4          # 变动值 (+/-)
balance_after  int4          # 变动后余额
source         text          # auto_settlement / redemption / admin
reason         text          # 变动原因说明
period         text          # 月份: 2026-07
created_by     text          # 操作人
created_at     timestamptz   # 时间
```

### redemption_orders — 兑换订单表
```
id             int8 PK       # 自增
uid            text FK       # 关联 kocs
discord_name   text          # 冗余
koc_name       text          # 真实姓名
option_type    text          # merch / diamonds / gplay
option_name    text          # 选项名称
points_spent   int4          # 消耗积分
reward_amount  text          # 奖励内容描述
contact_info   text          # 联系信息
status         text          # pending / shipped / cancelled
period         text          # 月份
admin_notes    text          # 备注
created_at     timestamptz   # 创建时间
processed_at   timestamptz   # 处理时间
```

## 八、核心业务流程

### 8.1 创作者完整生命周期

```
入群 → Mochi自动私信 → 注册填写信息 → 新人月投稿 
  → 满5分+2bonus → 连续创作 → 升级金级 
  → 持续投稿 → 升级铂金 → 兑换奖励
```

### 8.2 月度结算流程 (完整闭环)

```
┌──────────────────────────────────────────────────────────┐
│                   月度结算流程                             │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ① 月底 (管理员点广播)                                    │
│     └─ Mochi 通知: "收集作品链接已开启，请在5号前提交"      │
│                                                          │
│  ② 截止日 (次月5号)                                       │
│     └─ 停止收集                                         │
│                                                          │
│  ③ 管理员操作:                                           │
│     ├─ 导出待审核 JSON                                    │
│     └─ 复制 Agent 提示词                                  │
│                                                          │
│  ④ 发给 Codex Agent                                     │
│     └─ Agent 用浏览器逐条查看链接互动数                     │
│     └─ 根据规则计算积分                                   │
│     └─ 输出 算分 JSON                                     │
│                                                          │
│  ⑤ 管理员导入 JSON                                       │
│     ├─ 粘贴 或 上传文件                                    │
│     └─ 预览 (自动合并同UID)                               │
│                                                          │
│  ⑥ 确认应用积分 (核心操作)                                 │
│     ├─ ✅ 标记 submissions 为 scored                      │
│     ├─ ✅ 写入 point_logs (积分到账创作者端)               │
│     ├─ ✅ 检测 CC: 15+条 → 自动创建端内奖励订单            │
│     └─ ✅ 检测新人: 新人月满5分 → 自动 +2 bonus           │
│                                                          │
│  ⑦ 管理员点广播                                          │
│     └─ "算分完成通知" → Mochi 通知所有创作者                │
│                                                          │
│  ⑧ 创作者操作                                            │
│     ├─ 登录查看积分                                       │
│     └─ 兑换奖励 (钻石/谷歌卡/周边)                        │
│                                                          │
│  ⑨ 管理员处理发货                                         │
│     └─ 周边/端内奖励/谷歌卡 → 标记已发货                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 8.3 等级晋升流程

```
认证创作官 (默认)
     │
     ├─ 连续2个月月积分≥5分 ──→ 🟡 金级创作官
     │                              │
     │                              └─ 连续3个月月积分≥5分 ──→ 💎 铂金创作官
     │
     └─ 未达标 → 保留当前等级
```
